Base64エンコード・デコードの使い方
【日本語・URLセーフ対応】

2026年5月8日 読了約4分 0-TOOLs

APIのトークン確認、メール添付のMIMEエンコード、画像のdata URI生成——Web開発では日常的にBase64が登場します。

0-TOOLsのBase64変換ツールは、テキストのエンコード・デコードをブラウザ内で処理するため、機密情報をサーバーに送信せず安全に変換できます。日本語（UTF-8）・URLセーフ形式にも対応しています。

Base64とは？

Base64は、バイナリデータや特殊文字を含むテキストを、英数字と記号64種類だけで表現するエンコード方式です。メールシステムやHTMLなど、ASCII文字しか扱えない環境でもデータを安全に伝送できます。

処理 方向 用途
エンコード テキスト → Base64文字列 APIトークン生成・data URI作成
デコード Base64文字列 → テキスト APIレスポンスの内容確認・JWTのペイロード確認

URLセーフ形式とは？

通常のBase64は「+」「/」「=」を使いますが、URLに含めると誤動作することがあります。URLセーフ形式では「+」を「-」、「/」を「_」に置き換えることで、URLパラメータや認証トークンに安全に使えます。

JWTについて JWT（JSON Web Token）のヘッダーとペイロードはURLセーフBase64でエンコードされています。このツールでデコードすると内容を確認できます（署名の検証は行いません）。

使い方（3ステップ）

  1. 1
    Base64変換ツールを開く ブラウザで即アクセスできます。処理はすべてブラウザ内で完結します。
  2. 2
    エンコード／デコードを選択してテキストを入力 URLセーフ形式が必要な場合はスイッチで切り替えます。
  3. 3
    結果をコピーする 変換結果をワンクリックでクリップボードにコピーできます。

活用シーン

  • Web開発：APIのBasic認証ヘッダー（username:passwordのBase64）の生成
  • JWT確認：JWTのペイロード部分をデコードして内容を確認
  • メール開発：MIMEエンコードの動作確認
  • データURI：小さな画像やフォントのBase64データURI生成
