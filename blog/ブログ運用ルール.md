## 1. 新記事の追加手順

### Step 1: 記事ファイルを作成する

`blog/sample-article.html` をコピーして `blog/(スラッグ)/index.html` として保存。

```
/blog/記事スラッグ/index.html
```

**スラッグの命名ルール:**
- 英数字小文字 + ハイフン区切り
- キーワードを含める
- 短く簡潔に（3〜5単語）
- 例: `pdf-compress-tips`, `color-code-guide`, `image-size-reduce`

### Step 2: ★マーク箇所を書き換える

`sample-article.html` 内の `★` コメントを検索して以下を更新:

| 項目 | 内容 |
|------|------|
| `<title>` | KW【補足KW】\| 0-TOOLs.JAPAN（30〜55文字） |
| `<link rel="canonical">` | 実際のURL |
| `<meta name="description">` | 100〜150文字の記事説明 |
| OGPタグ（og:title等） | title・description・image・url |
| JSON-LD | headline・description・datePublished・url |
| パンくずリスト | 最後のspanを記事名に |
| `.article-category` | カテゴリ名 |
| `.article-title` | H1タイトル |
| `.article-meta` | 公開日・読了時間 |
| アイキャッチ | プレースホルダーを画像タグに差し替え（任意） |
| 本文 | H2/H3/H4 構造で執筆 |
| CTA | ツールへのリンク |
| 関連記事 | 関連ページへのリンク |

### Step 3: ブログ一覧に追加する

`blog/index.html` の `<div class="blog-grid">` 内に記事カードを追加する。

**テンプレート（一覧の先頭に追加）:**

```html
<a class="blog-card" href="/blog/(スラッグ)/" data-category="(カテゴリ)">
  <div class="blog-card-img-placeholder">📝</div>
  <div class="blog-card-body">
    <span class="blog-card-category">カテゴリ名</span>
    <div class="blog-card-title">記事タイトル</div>
    <p class="blog-card-desc">記事の説明文（2〜3行）</p>
    <div class="blog-card-meta">
      <span>2026年M月D日</span>
      <span>読了N分</span>
    </div>
  </div>
</a>
```

**data-category の値（フィルター用）:**

| 値 | カテゴリ名 |
|----|-----------|
| `pdf` | PDF活用 |
| `image` | 画像変換 |
| `web` | Web制作 |
| `guide` | 使い方ガイド |

### Step 4: sitemap.xml を更新する

`sitemap.xml` のブログ記事セクションに追記:

```xml
<url>
  <loc>https://0-tools.com/blog/(スラッグ)/</loc>
  <lastmod>YYYY-MM-DD</lastmod>
  <changefreq>monthly</changefreq>
  <priority>0.7</priority>
</url>
```

## 2. SEOタイトルの付け方

### 基本フォーマット

```
主要KW【補足KW】| 0-TOOLs.JAPAN
```

### 良いタイトルの例

- `PDFをJPEGに変換する方法【無料・登録不要】| 0-TOOLs.JAPAN`
- `画像をWebPに変換する3つの方法【ブラウザ完結】| 0-TOOLs.JAPAN`
- `カラーコードをHSLに変換する方法【コピペOK】| 0-TOOLs.JAPAN`

### タイトル作成チェックリスト

- [ ] 30〜55文字に収める
- [ ] 主要キーワードをタイトル前半に含める
- [ ] 「無料」「登録不要」「簡単」など便益ワードを入れる
- [ ] 「| 0-TOOLs.JAPAN」を末尾に付ける
- [ ] 競合と差別化できる切り口があるか確認

### NGパターン

- ❌ 長すぎる: `PDFファイルをJPEG画像に変換する方法を3つ紹介【初心者でも簡単・無料・登録不要・ブラウザだけ】| 0-TOOLs.JAPAN`
- ❌ キーワードなし: `画像フォーマットについて | 0-TOOLs.JAPAN`
- ❌ 誇大: `最強・最高・神ツール`

## 3. おすすめ記事構成

### 基本テンプレート（How-to記事）

```
H1: [方法名]【[状況/条件]】
└ リード文（なぜこれが必要か・何ができるか・2〜3段落）
└ 目次
├ H2: [対象物]とは？（基礎知識）
├ H2: [対象物]の[メリット/特徴]N選
│   ├ H3: ① 〜
│   ├ H3: ② 〜
│   └ H3: ③ 〜
├ H2: [方法名]の手順（N ステップ）
│   ← step-list クラスで番号付きリスト
│   ← CTAボックスをここに配置
├ H2: [注意点/よくある失敗]
└ H2: まとめ
   └ 関連記事セクション
```

### 文字数の目安

- 最低 800文字（薄い記事はGoogleに評価されにくい）
- 理想 1,500〜2,500文字（網羅性が高い記事は上位表示されやすい）
- 読了時間の目安: 500文字/分で計算

### よく使う要素

| 要素 | クラス | 用途 |
|------|--------|------|
| 注意書き | `.note-box` | 補足・注意事項 |
| 手順リスト | `.step-list` | 手順説明 |
| 比較表 | `.compare-table` | フォーマット比較など |
| CTAボックス | `.cta-box` | ツールへの誘導 |
| コードブロック | `<pre><code>` | コード表示 |

## 4. 内部リンクルール

### 必須リンク

各記事に以下を必ず含める:

1. **関連ツールへのリンク**（1〜2個）
   - 記事本文中に自然な文脈で埋め込む
   - 関連記事セクションにも `.related-card` として追加

2. **ブログ一覧へのリンク**
   - パンくずリストの「ブログ」リンクで自動対応

3. **関連記事へのリンク**（2〜4個）
   - `.related-section` の `.related-grid` に追加

### リンクテキストのルール

- ✅ 具体的なKW: `画像変換ツール`、`PDF→JPEG変換ツール`
- ✅ 記事名そのまま: `PDFをJPEGに変換する方法`
- ❌ 汎用テキスト: `こちら`、`詳細はこちら`、`ここ`

### ツールページへのリンクパス

| ツール | パス |
|--------|------|
| 画像変換 | `/tools/img-converter/` |
| PDF→JPEG | `/tools/pdf/pdf-to-jpeg/` |
| PDF圧縮 | `/tools/pdf/pdf-compress/` |
| カラーコンバーター | `/tools/collar/collar-converter/` |
| 日数カウンター | `/-Day-Counter/` |

## 6. OGP画像の作成ルール

OGP画像はソーシャルシェア時に表示される画像です。

- サイズ: **1200 × 630px**
- ファイル形式: PNG または JPEG
- 保存先: `assets/images/blog/(スラッグ)-og.png`
- 内容: 記事タイトル + 0-TOOLs.JAPANのロゴ/ブランド

OGP画像がない場合は `og:image` タグを削除するか、サイト共通のデフォルト画像を指定してください。

---

## 7. アイキャッチ画像の運用

アイキャッチ画像がない場合は、テンプレートの絵文字プレースホルダーを使用できます:

```html
<div class="article-thumb-placeholder" role="img" aria-label="説明">🖼️</div>
```

画像がある場合は `<img>` タグに差し替え:

```html
<img
  class="article-thumb"
  src="../../assets/images/blog/(スラッグ).webp"
  alt="画像の説明（alt text）"
  width="740" height="416"
  loading="eager"
  itemprop="image"
>
```

- フォーマット: **WebP推奨**（ページ速度向上のため）
- サイズ: 740 × 416px（16:9）
- ファイル保存先: `assets/images/blog/`
